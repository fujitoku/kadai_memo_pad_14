# ①課題名
**メモ帳（3ページ構成）**

## ②課題内容（どんな作品か）
シンプルで機能的なメモ帳

### 主な機能
- **メモ作成ページ**: 今日の日付を表示し、タイトルと内容を入力してメモを作成
- **カレンダーページ**: 月単位のカレンダーでメモを視覚的に管理、日付ごとにメモを確認
- **アクティビティページ**: メモの追加履歴をタイムライン形式で表示、期間フィルター機能付き

### 技術スタック
- **HTML5**
- **CSS**: Tailwind CSS 
- **JavaScript**: jQuery 2.1.3
- **データ管理**: LocalStorage（ブラウザ内でデータ永続化）

### デザイン
- 白黒のミニマルデザイン
- モダンなボタンアニメーション（ホバーエフェクト、トランジション）
- グラスモーフィズム風のUI要素
- レスポンシブ対応（モバイル/タブレット/デスクトップ）

## ⑤工夫した点・こだわった点
- **ボタンアニメーション**: ホバー時の浮き上がり効果
- **日付ベースのデータ管理**: メモを日付ごとに整理し、カレンダービューで視覚的に確認可能
- **アクティビティログ**: メモの作成履歴を時系列で表示し、期間フィルター（今日/今週/今月/すべて）で絞り込み可能


## ⑥難しかった点・次回トライしたいこと（又は機能）

### 難しかった点
- **LocalStorageのデータ構造設計**: メモとアクティビティの2種類のデータを効率的に管理する構造の検討
- **カレンダーの動的生成**: 月の最初の曜日や日数を計算し、正しくグリッド配置する実装
- **相対時刻の計算**: タイムスタンプから「○分前」「○時間前」を計算するロジック
- **CSS統一**: Tailwind CSSとカスタムCSSの共存、スタイルの一貫性を保つこと

### 次回トライしたいこと
- **メモの編集機能**: 既存のメモを後から編集できる機能
- **カテゴリー/タグ機能**: メモを分類して管理
- **検索機能**: タイトルや本文から全文検索
- **ダークモード切り替え**: 白黒デザインとダークモードの切り替え
- **背景画像対応**: ユーザーが好きな画像を背景に設定できる機能

## ⑦フリー項目（感想、シェアしたいこと等なんでも）

### 感想
- **シンプルさの追求**: 白黒デザインに統一することで、情報の視認性が向上し、ユーザーがメモの内容に集中できるUIになった
- **LocalStorageの便利さ**: サーバー不要なこと

### 学んだこと
- LocalStorageの制限
- カレンダーアプリケーションにおける日付計算のロジック
- モバイルファーストなレスポンシブデザインの考え方

- index.htmlでメモを保存 → ローカルストレージに保存
- calendar.htmlを開く → 同じローカルストレージから読み取り
- activity.htmlを開く → 同じローカルストレージから読み取り
